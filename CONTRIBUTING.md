# 贡献指南

> 虽然我们非常感谢大量的拉取请求和社区参与，但一些拉取请求是专门为了推广提供付费解决方案的公司API而开放的。这个API列表不是营销工具，而是帮助社区快速轻松地构建应用程序和使用免费公共API的工具。被识别为营销尝试的拉取请求将不会被接受。
>
> 请确保您要添加的API具有完全免费访问权限或至少有免费层级，并且不依赖于在提交之前购买设备/服务。一个会被拒绝的例子是用于控制智能插座的API - API是免费的，但您必须购买智能设备。
>
> 感谢您的理解！ :)

## 格式要求

当前API条目格式：

| API | 描述 | 认证 | HTTPS | CORS |
| --- | --- | --- | --- | --- |
| API标题（链接到API文档） | API的描述 | 此API是否需要身份验证？* | API是否支持HTTPS？ | API是否支持[CORS](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/CORS)？* |

示例条目：

```
| [NASA](https://api.nasa.gov) | NASA数据，包括图像 | No | Yes | Yes |
```

\* 目前，`认证`字段只接受以下输入：

* `OAuth` - 需要OAuth认证
* `apiKey` - 需要API密钥
* `X-Mashape-Key` - 需要X-Mashape-Key头
* `User-Agent` - 需要User-Agent头
* `No` - 无需认证

\* 目前，`CORS`字段只接受以下输入：

* `Yes` - 支持CORS
* `No` - 不支持CORS
* `Unknown` - CORS支持状态未知

## 提交新API的要求

在提交新API之前，请确保：

1. **免费访问**：API必须提供免费访问或至少有免费层级
2. **公开可用**：API必须是公开可用的，不需要特殊权限或设备
3. **文档完整**：API必须有完整的文档
4. **稳定性**：API应该是稳定的，不是实验性的
5. **合法性**：API的使用必须是合法的

## 中文化特殊要求

由于这是中文版本，我们还有以下额外要求：

1. **中文描述**：API描述必须翻译成准确的中文
2. **术语统一**：使用统一的技术术语翻译
3. **文化适应**：考虑中文用户的使用习惯
4. **本土API优先**：优先收录对中文用户有用的API

## 分类指南

请将API添加到正确的分类中。如果不确定分类，请：

1. 查看现有类似API的分类
2. 选择最符合API主要功能的分类
3. 如果需要新分类，请在Issue中讨论

## 排序规则

在每个分类中，API按照以下规则排序：

1. 按API名称的字母顺序排列（忽略大小写）
2. 中文API名称按拼音排序
3. 数字在字母之前

## 提交流程

1. **Fork仓库**：Fork这个仓库到您的GitHub账户
2. **创建分支**：为您的更改创建一个新分支
3. **添加API**：按照格式要求添加新的API
4. **测试链接**：确保所有链接都是有效的
5. **提交PR**：提交拉取请求并详细描述您的更改

## 质量检查

在提交之前，请确保：

- [ ] API链接有效且指向正确的文档
- [ ] 描述准确且简洁（不超过100个字符）
- [ ] 认证、HTTPS、CORS字段填写正确
- [ ] 按字母顺序正确插入
- [ ] 中文翻译准确

## 报告问题

如果您发现：

- 无效的API链接
- 不准确的信息
- 翻译错误
- 格式问题

请通过[Issues](https://github.com/public-apis-zh/public-apis-zh/issues)报告。
